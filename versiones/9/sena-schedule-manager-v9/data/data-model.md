    # SENA Schedule Manager - Data Model
    This document defines the logical data model for the MVP system based on MongoDB.
    Collections are named in English singular form as required.
    The model supports room, training group, instructor, schedule block, and observation entities.
    Conflict detection is handled via indexes on scheduleBlock for overlapping time intervals per room, group, or instructor.
    PII fields are identified for instructors and observation authors.
    Retention rules keep data for 5 years after deactivation (or cancellation) for legal/audit purposes.

          required: false

          required: true

        - name: created_at

          required: true
